General Form Accessibility Guidlines

Form elements are generally already accessible to screen readers as they use semantic markup. Most of the necessary ARIA markups are automatically injected in by javascript. For some components, and application page layouts, some hard-coding is necessary to make them accessible.

Labels for Individual Form Elements

Each form element, i.e. inputs, textareas, selects, checkboxes, radio buttons, is typically associated with a label. There are two ways to associate a label to a form element:

  1. Explicitly by giving the form element an [id="*"], and giving the label a [for="*"], where * designates identical values. This is common where you have the label in a separate grid from the form element.
  2. Implicitly by placing the form element inside the label. In this case you will not need to apply an ID and For attribute. This is common when using radio buttons and checkboxes.

In browsers supporting HTML5, clicking the label will cause the corresponding form element to gain focus.

Labels for Grouped Form Elements

                    
In some situations, you may use a label to describe a group of elements i.e. you may have a label in one grid describing a collection of inputs in an adjacent grid. In this case a label does not need a for attribute linked to a specific form id. Instead, the label itself will need an id.

                    
Using Icons

                    
When using icons in the place of text for buttons, links etc, ensure that a title attribute is added to the control so screen readers can pick up on the function. By default, elements with title attributes cause a dark tooltip to appear. To prevent this occuring, you can add a .int-noTooltip.

Input and Text Area

                        To turn the text area into a multiline text area, add the [rows="*"] where * designates a positive integer.
                    

                    

                        
Radio Buttons

                        
Ensure the div.int-grid wrapping the radio buttons contains a .int-radioGroup and the first div.int-grid contains the label for the group.

                        
Checkboxes

                        
Ensure the div.int-grid wrapping the checkboxes contains a .int-checkboxGroup and the first div.int-grid contains the label for the group.

                        
Multiple Inline Inputs

                        
Ensure that the div.int-row containing the multiple inputs has a .int-inputGroup and the first div.int-grid contains the label for the group.

You can use standard MVC validation messages as normal. The validation message must be the last item in the row and must be outside the grids.

                        
@* Required form fields *@
using (Html.BeginRow())
{
    using (Html.BeginGrid(3))
    {
        @Html.OuLabelFor(m => m.FormInputRequired, "Required input", "This is required", required: true)
    }
    using (Html.BeginGrid(5))
    {
        @Html.OuTextBoxFor(m => m.FormInputRequired)
    }
    @Html.ValidationMessageFor(m => m.FormInputRequired)
}
@* End of required form fields *@

@* Readonly and disabled form fields *@
@Html.OuTextBox("ReadOnlyInputTextBox", isReadonly: true)
@Html.OuTextBox("DisabledInputTextBox", isDisabled: true)
@* End of readonly and disabled form fields *@
                		

                    

                    

                        
Required Field

                        

                            * denotes required field must be included at the top of any form containing 'required' fields.
                        

                        

                            label - include a span containing * with a class of int-requiredAsterisk.
                        

                        

                            input - include an attribute of required.
                        

                        

                            HTML5 attributes do not require an attribute value i.e. required versus required="true"
                        

                        
Validation

                        

                            To render a validation error inject a class of int-error in to int-row to form int-row int-error this adds the pink background colour to the row. Add the error message itself by including a span with a class of int-errorMessage and ensure the span with message is located before the the closing int-row tag.
                        

                        

                            Error messages should not be placed directly within an int-grid.
                        

                        
Read only and Disabled Form Fields

                        

                            Add a readonly attribute to any input to change state to read only.
                        

                        

                            Add a disabled attribute to any input to disable.
                        

                        

                            For data that is read only within a form but that will never be editable use Read only plain text.
                        

                    

                    

                        
Validation

                        
Each span.int-errorMessage requires unique ID.

                        

                            An input with validation error must include [aria-describedby="*"], where * is the unqiue ID of span.int-errorMessage.

The check all checkbox allows users to select a group of check boxes with one selection. Create a series of checkboxes. Add class of .int-checkAll to the first checkbox in the series. Give the entire series of checkboxes the same name attribute value. As you can see from the example, only the checkboxes with the same name will be affected by the selection of the .int-checkAll checkbox.

                    

                    

                        
Ensure the div.int-grid wrapping the checkboxes contains a .int-checkboxGroup and the first div.int-grid contains the label for the group. The label for the group requires an unique ID.

Associated

                        

                            For associated buttons, if the button relates to an input control, then an attribute of [aria-controls="*"] needs to be added to the button with * being the ID of the input.

using (Html.BeginRow(true))

    using (Html.BeginGrid(5))

For instance if search is the first feature at the top of an application page, using a large search gives the feature prominence in a similar way to that of search engines, who use large search components to capture the initial search query.
                        

                        

                            Search inputs require a type with a value of search.
