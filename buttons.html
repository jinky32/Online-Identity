Drop Down Menu

                        
A drop down menu is composed of a button, followed by an unordered list.int-dropMenu.

                        
Each list item will correspond to a menu item.

                        

                            Extras A divider between list items can be created by adding an empty list item of .int-dropMenuDivider.
                        

                    

                    

                        

                            
                                
                                
                            
                            
                                

                                    Version
                                    Changes
                                

                            
                            
                                

                                    v3
                                    Drop Down Menu, and Drop Up Menu, merged so there is only one Drop Menu component. The position of the drop menu is dynamic so will display up of down or to the side depending on the viewport. The new markup has no container, only a button and a menu. The button has a class of .int-dropMenuButton.
                                

                            
                        

                    

                
       
                
                
                
                
Drop Content

                

                    

Drop Content 


    Drop Content


                    

                        

                        
Razor

                        
HTML

                        
Supporting Text

                        
Changelog

                    

                    

                    

                        
The content to be toggled must be wrapped in @<text> ... </text> to allow multi-line markup content to be used. Content without markup can be passed in as a string.

                        
@HTML.DropContent("Drop Content", 
    @<text><p>Here is the content</p>
      <p>Here is some more content</p>
      <p>Hello @User.Identity.Name</p></text>)

@HTML.DropContent("Drop Content", "Here is the content")
                

                    

                    

                        
A drop content is a cross between a drop menu (above) and a content toggler. The drop content is used to toggle hidden content like a content toggler but the content is taken out of the normal flow of the document (i.e. the content appears above current elements on the page), so its appearance is similar to a drop menu.

                        
A drop content is made up of a trigger button and a content section: create an a.int-button.int-dropDivButton, for the trigger; and a div.int-dropDiv for the content section. 

                        
The content section will span the full length of its parent element.

                                    Changes
                                

                            
                            
                                

                                    v3
                                    Container div.int-dropDownContent removed. Button now contains new class .int-dropDivButton. div.dropContent replaced with div.int-dropDiv.

@HTML.CheckAllDropMenu(
    HTML.DropMenuItem("Edit", "/", icon:"pencil"), 
    HTML.DropMenuItem("Delete", "/", icon:"trash")
)
                

                    

                        
To create a check all button with drop down menu options, first create a div.int-checkMenu.int-dropDown containing:

                        
    
                            
  • a button containing checkbox of .int-checkAll;
    • 
                            
  • a button containing a i.int-icon-claret-down (this will be the drop down menu button);
    • 
                            
  • an unordered list of .int-dropMenu, populated with the list items you want for your menu options. 
    • 
                        

                        
Next, add in the number of checkboxs you require, wrapped around div tags (this will stack the inputs; to display inline, omit the div tags).

                        
Lastly, give each of the checkboxes a matching [name="*"] attribute value.

Radio Switches

                

                    Radio switches display as typical radio buttons and checkboxes in IE8 and below.
                

                

                    



    

        
    

    

            
            
            
            
            
            
            
            
            
            
            
            
            
            
    



                    

                    

                        
Razor

                        
HTML

                        
Supporting Text

                        
Changelog

                    

                    

                    

                        
RadioSwitch accepts between 2 and 5 items and will apply the appropriate class based on the number of items passed in.

                        
@HTML.RadioSwitch("Frequency", 
    HTML.RadioSwitchItem("day", "radioSwitch1", "Day"), 
    HTML.RadioSwitchItem("week", "radioSwitch1", "Week"), 
    HTML.RadioSwitchItem("month", "radioSwitch1", "Month")
)
                

                    

                    

                        
Create a div with .int-radioSwitch. Add an additional class depending on how many radio buttons you wish to have:

                        
    
                            
  • For two radio buttons, no additional class is added
    • 
                            
  • For three radio buttons, add the class .int-radioSwitchThree
    • 
                            
  • For four radio buttons, add the class .int-radioSwitchFour
    • 
                            
  • For five radio buttons (the maximum), add the class .int-radioSwitchFive
    • 
                        

                        
Within the div.int-radioSwitch, create an input[type="radio"] followed by a label for each of the items. 
                            Each radio button should have an unique id, and all the radio buttons within the group should share the same name. 
                            The first radio button should have an attribute of [checked]. The label for each radio should have a [for="id"] that matches the id of the corresponding radio.
 
                        
The final step is to add a span.int-slideButton which creates the bar for sliding between radio buttons.

The content to be toggled must be wrapped in @<text> ... </text> to allow multi-line markup content to be used. Content without markup can be passed in as a string.

                        
@HTML.ContentToggler("Show/Hide", 
    @<text><p>Here is the content</p>
      <p>Here is some more content</p>
      <p>Hello @User.Identity.Name</p></text>)

@HTML.ContentToggler("Show/Hide", "Here is the content")
                		

                    

                    

                        

                            This is a generic content toggler which can be used to show hide any element on the page (in the above example we are hiding a form feature). 
                        

                        

                            Add .int-toggle to any element to hide the element. Add an a.int-button immediately preceding your hidden element to toggle it into view - 
                            this order is crucial as the JavaScript looks for the preceeding sibling button to act as the trigger for the hidden element.

The content to be toggled must be wrapped in @<text> ... </text> to allow multi-line markup content to be used. Content without markup can be passed in as a string.

                        
@HTML.ToggleAllToggler("Show/Hide All", 
    HTML.ToggleSection("First Section",
        @<text><p>Here is the first section of content</p>
            <p>Here is some more content</p>
            <p>Hello @User.Identity.Name</p>
        </text>),
    HTML.ToggleSection("Second Section",
        @<text><p>Here is the second section of content</p>
            <p>Here is some more content</p>
            <p>Hello @User.Identity.Name</p>
        </text>),
    HTML.ToggleSection("Third Section",
        @<text><p>Here is the third section of content</p>
            <p>Here is some more content</p>
            <p>Here is even more content</p>
            <p>Here is more content still</p>
            <p>Hello @User.Identity.Name</p>
        </text>)
)
                		

                    

                    

                        

                            The content toggler uses a div.int-toggler which contains a series of h3+div. The h3 creates the toggler header/trigger and the div holds the hidden content. Optionally add a class of int-togglerContentActive to the div you want visible by default.
                        

                        

                            Two types of buttons can be used with the toggler. One allows you to show all the divs of a all instances of togglers on a page, and the second allows you to show all the divs of a single (the next) instance of the toggler.
                        

                        

                            To show all instances, add a button anywhere on the page with the class int-showAllToggler. To show the next instance, place a button directly before the toggler instance and with class int-showToggler.

Individual Checkbox Buttons









                    

                    

                        
HTML

                        
Supporting Text

                        
Changelog

                    

                    

                    

                
Switch buttons are basically radio buttons or checkboxes that have been styled to look like buttons.

                
For groups of radio switch buttons, add a .int-radioSwitchButton to the div which contains the radio buttons and labels.

                
For groups of checkbox switch buttons, add a .int-checkboxSwitchButton to the div which contains the checkboxes and labels.

                
For the group of switch buttons, the HTML markup is minimal as the for attribute and ID of the labels and inputs respectively, are added via JS.

                
For individual checkbox buttons, ensure the input has a int-checkboxButton. 

                
The individual checkbox buttons comes in two configurations, one where the checkbox is visible inside the button, and one where the checkbox is hidden. 

                
To display the checkbox, nest the input inside the label. To hide the input, add a int-hide to the input and ensure the input is nested outside of the label.

                
The id and for attributes need to be hardcoded for individual checkbox buttons.

                
Add a class of int-selected to the label if you want a default button to be selected initially. For radio switch buttons, this can only be added to one label. For checkboxes, this can be added to any number of labels.
