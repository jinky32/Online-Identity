How To Use Icons

There is known cross-domain issue when testing fonts in Firefox. The issue only affects your local test environment. To modify your local environment do the following: Go to IIS Manager > HTTP Response > Add. Under 'Name' enter: Access-Control-Allow-Origin, then under 'Value' enter: *.

Then, whilst in IIS Manager open MIME Types > Add. In 'File Name Extension' add: .woff and in 'MIME Type' add: font/x-woff. Then restart IIS.

Icon library provided by Font Awesome (v.3.1.2)

Icons can be used either on their own, as buttons or as button with text:

Icons are ideally used to communicate a function with the least number of words, or even no words at all which is particularly useful where space is limited.

Icon buttons take the same parameter options as buttons but with an extra icon parameter. To create an icon only button, pass null for the displayText parameter.

                        
@HTML.Icon("star")
@HTML.IconButton(null, "dashboard", "/")
@HTML.IconButton("Button with tooltip and id", "beer", "/", id:"ButtonWithTooltip", title:"This button has a tooltip")
                		

                    

                    

                        

                            Icons are defined using an i with a class name (see list of icons below with displayed class name, such as int-icon-plus).
                        

                    

                    

                        

                            If using icons within a button or link without supporting text, the title attribute is used within the anchor or button to help screen readers determine the button's function.
