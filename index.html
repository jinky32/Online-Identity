UI Framework Site Structure

<div id="int-site">
    <nav id="int-primary"></nav>

    <div id="int-page">
        <div id="int-region1"></div>

        <div id="int-region2">
            <div id="int-content"></div>
        </div>
    </div>
</div>

The site structure of the UI Framework is made up of the main site area, which contains the top menu (primary navigation) and the page area. The primary navigation lists business focus areas i.e. Curriculum, Permissions, Utilities.

The page area contains the sidebar (secondary navigation) and the content region. The content region contains the main content area. The sidebar is used to display applications and application pages i.e. Credit Transfer Claim. The main content area displays application content.

The main content area by default is a one column layout. This can be changed to a two or three column layout. See Columns for more detail. Each column can be further divided into rows, and each row can be divided up into grids. See Rows & Grids for more details.

CSS and style syntax

The framework uses a reserved classname and ID prefix of int-. Only standard components within the framework may use int-. If custom styles are created within your application it is advised you create your own prefix to avoid conflicting styles.

CSS syntax key and how it is used within the UI.Docs site.

Supporting Text Key
CSS Syntax HTML Equivalent Description
.abc class="abc" class of "abc"
#abc id="abc" id of "abc"
div.abc <div class="abc"></div> div with class of "abc"
div#abc <div id="abc"></div> div with id of "abc"
div.abc.def <div class="abc def"></div> div with classes "abc" and "def"
div#abc.def <div id="abc" class="def"></div> div with id of "abc" and class of "def"
a.abc[href="#def"] <a class="abc" href="#a"></a> anchor with class of "abc" and a href attribute with a value of "#def"
input.abc[type="text"] <input class="abc" type="text" /> input with class "abc" and a type attribute with a value of "text"
a.abc[href="#def"][title="hello"] <a class="abc" href="#def" title="hello"></a> anchor with class of "abc", a href attribute with a value of "#def", and a title attribute with a value of "hello"

Adding your own Classes and IDs

You can create your own classes if necessary. It is advisible that you set and follow your own naming conventions i.e. .permissionsPaddingLeft or .curriculumPaddingRight.

To ensure consistency across applications, it is advised that you use the default UI styles as much as possible.
