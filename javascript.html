Javascript

The framework provides a number of javascript functions/plugins that can be called from within your client code. The most useful of these are summarised below.

Aria Helper (js/uif-global.js)

If you make any changes to the DOM via Javascript, you must ensure that aria has been updated across any new elements. Fortunately, the framework provides some handy helpers to do all of this for you. All you have to do is pick the right one! You may call as many of these as you require depending on the DOM changes you have made. They all run across the entire DOM so try not to call the ones you don't need.

Javascript
// Adds aria to main structure elements and click event to Skip To Main Content links
OU.AppFramework.AriaHelper.UpdateMain();

// Adds aria to buttons, wires up drop down/up/content menu buttons, initialises content togglers, more/less info buttons, etc.
OU.AppFramework.AriaHelper.UpdateButtons();

// Adds aria to checkboxes and radio buttons. Wires up watermarking to text boxes. Wires up help functionality.
OU.AppFramework.AriaHelper.UpdateForms();

// Adds aria to breadcrumb, jumplinks and pagination
OU.AppFramework.AriaHelper.UpdateNavigation();

// Adds aria to alerts. Wires up tooltips to elements with title set. Wires up alert close button click event.
OU.AppFramework.AriaHelper.UpdateNotifications();

// Initialises sortable and expandable tables. Wires up click toggling of child tables.
OU.AppFramework.AriaHelper.UpdateTables();

// Initialises data tables
OU.AppFramework.AriaHelper.UpdateDatatables();

// Adds aria to lists, icons and notes
OU.AppFramework.AriaHelper.UpdateTypography();

Alerts plugin (js/widgets/jquery.alert.js)

To display alert notifications within a container from client-side code, the framework provides an alert plugin that can be called against the jquery selector for the container that will hold the alert. You must call the Aria Notification straight afterwards to apply the necessary aria to the markup.

Javascript
// To display a sticky error alert that is all bold
$('#message-container').alert({ title: 'One or more errors were detected on this page. Please review now.', sticky: true, level: 'error' });
OU.AppFramework.AriaHelper.UpdateNotifications();

// To display an info alert
$('#message-container').alert({ title: 'Standout text', text: 'Normal text that appears next.', level: 'info' });
OU.AppFramework.AriaHelper.UpdateNotifications();

// To display a success alert without a title
$('#message-container').alert({ text: 'Something was successful by all accounts.', level: 'success' });
OU.AppFramework.AriaHelper.UpdateNotifications();

// To display a standard alert
$('#message-container').alert({ text: 'This alert has no extra style.' });
OU.AppFramework.AriaHelper.UpdateNotifications();

Utility Helper (js/uif-global.js)

Utility functions for altering page markup post-load. Includes functions for applying validation styles and recalculating the height of the scrollbar if a page level control bar is being used.
NB: These functions are automatically called during page load within the main Layout page. Only explicitly call them if your pages perform any Javascript-specific rendering.

Javascript
// To apply validation styles
OU.AppFramework.UtilityHelper.ApplyValidationStyles('#int-content');

// To recalculate the height of the page control bar, which stops the scrollbar disappearing behind it
OU.AppFramework.UtilityHelper.CalculatePageControlBarHeight();
